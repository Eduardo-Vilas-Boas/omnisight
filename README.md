# omnisight

The 'omnisight' project was generated by using the default-python template.

* `src/`: Python source code for this project.
  * `src/omnisight/`: Shared Python code that can be used by jobs and pipelines.
* `resources/`:  Resource configurations (jobs, pipelines, etc.)
* `tests/`: Unit tests for the shared Python code.
* `fixtures/`: Fixtures for data sets (primarily used for testing).

# Using this project using the CLI

The Databricks workspace and IDE extensions provide a graphical interface for working
with this project. It's also possible to interact with it directly using the CLI:

1. Authenticate to your Databricks workspace, if you have not done so already:
    ```
    $ databricks configure
    ```

2. To deploy a development copy of this project, type:
    ```
    $ databricks bundle deploy --target dev
    ```
    (Note that "dev" is the default target, so the `--target` parameter
    is optional here.)

    This deploys everything that's defined for this project.
    For example, the default template would deploy a pipeline called
    `[dev yourname] omnisight_etl` to your workspace.
    You can find that resource by opening your workpace and clicking on **Jobs & Pipelines**.

3. Similarly, to deploy a production copy, type:
   ```
   $ databricks bundle deploy --target prod
   ```
   Note the default template has a includes a job that runs the pipeline every day
   (defined in resources/sample_job.job.yml). The schedule
   is paused when deploying in development mode (see
   https://docs.databricks.com/dev-tools/bundles/deployment-modes.html).

4. To run a job or pipeline, use the "run" command:
   ```
   $ databricks bundle run
   ```

5. Finally, to run tests locally, use `pytest`:
   ```
   $ uv run pytest
   ```

## MLflow tracking and deployment

The anomaly detector training script now logs hyperparameter experiments to MLflow,
including params, metrics, run artifacts, and a deployable best-model artifact.

1. Install dependencies:
    ```
    $ uv sync --dev
    ```

2. Activate virtual env
    ```
    $ source .venv/bin/activate
    ```

3. Get the 3D ConvNets Models
    ```
    cd src/omnisight/ingest/anomaly_detector/pytorch-resnet3d

    wget https://dl.fbaipublicfiles.com/video-nonlocal/i3d_nonlocal_32x2_IN_pretrain_400k.pkl -P models/
    ```

4. Convert these weights from caffe2 to pytorch
    ```
    python utils/convert_weights.py models/i3d_nonlocal_32x2_IN_pretrain_400k.pkl models/i3d_r50_nl_kinetics.pth
    ```

3. Run training with MLflow tracking:
    ```
    $ python src/omnisight/ingest/anomaly_detector/train_model.py \
         --dataset-source=local \
         --local-dataset-root=/path/to/IPAD_dataset \
         --mlflow-experiment-name="industrial-padim-anomaly-detector"
    ```

4. Optionally set a remote tracking backend and register the best model:
    ```
    $ python src/omnisight/ingest/anomaly_detector/train_model.py \
         --dataset-source=production \
         --dataset-root=/path/to/dataset \
         --mlflow-tracking-uri=http://localhost:5000 \
         --mlflow-register-model-name=industrial-padim-best
    ```
